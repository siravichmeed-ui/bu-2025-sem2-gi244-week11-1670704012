3.2 Now it's time for enemy to fight back! We are going to make the enemy move towards the player, and when it hit the player, it will apply a force to the player to push it away. Go to `Enemy.cs` script, and see we already have 3 variables: speed, rb, and player.

Let assign value to player and rb in the `Awake` method, and then add code to move the enemy towards the player in the `Update` method.

```c#
    void Awake()
    {
        player = GameObject.Find("Player");
        rb = GetComponent<Rigidbody>();
    }
```

```c#
    void Update()
    {
        var dir = player.transform.position - transform.position;
        dir = dir.normalized;
        rb.AddForce(dir * speed);
    }
```

**NOTE:** the interseting thing is how we calculate the direction from the enemy to the player, we just simply subtract the position of the enemy (SOURCE) from the position of the player (TARGET), and then `NORMALIZE` it to get the direction. This is a common way to calculate the direction from one object to another in Unity. the QUESTION is, why we need to normalize the direction? What will happen if we dont normalize it? (Hint: try to remove the `dir = dir.normalized;` line and see what happen)

## Power Up

4.0 In the scene, we have a Power Up object, which is a Diamond shape object at the center of the island. We are going to make it so that when the player hit the Power Up, it will give the player power to blast the enemy away. To achieve this, we need to add a trigger collider to the Power Up object, and then add code to detect when the player hit the Power Up, and then apply a force to the enemy to blast it away.

4.1 At the Power up object, notice that its collider is set to "Is Trigger", which means it will not collide with the player, but it will trigger an event when the player enter its collider. And its tag is set to "PowerUp", which we will use to detect when the player hit the Power Up.

In `PlayerController.cs` script, we are going to add `OnTriggerEnter` method to detect when the player hit the Power Up, and then apply a force to the enemy to blast it away. And a variable `hasPowerup` to check if the player has the power up or not.

```c#
public class PlayerController : MonoBehaviour
{
    public float speed = 5f;

    ...
    ...

    public bool hasPowerup = false;

    ...
    ...

    void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("PowerUp"))
        {
            hasPowerup = true;
            Destroy(other.gameObject);
        }
    }

    ...
    ...
}
```

4.2 Then we are adding KNOCK BACK force to the enemy when the player hit the enemy, but only when the player has the power up. So we need to check if the player has the power up or not before applying the force to the enemy.

```c#
    void OnCollisionEnter(Collision other)
    {
        if (other.gameObject.CompareTag("Enemy") && hasPowerup)
        {
            var enemyRb = other.gameObject.GetComponent<Rigidbody>();
            Vector3 awayFromPlayer = other.transform.position - transform.position;
            enemyRb.AddForce(awayFromPlayer * 5f, ForceMode.Impulse);
            Debug.Log($"Player collided with {other.gameObject.name} and has powerup");
        }
    }
```

in this code, the focus point is how we calculate the direction to apply the force to the enemy, we simply subtract the position of the player (SOURCE) from the position of the enemy (TARGET) to get the direction from the player to the enemy, and then we apply the force in that direction to blast the enemy away. The `ForceMode.Impulse` is used to apply an instant force to the enemy, which will make it blast away immediately when hit by the player with power up.

**NOTE: challenge student to think what if we want to apply a different force based on player's current speed when hitting the enemy** -- we could achieve this by multiplying the `awayFromPlayer` with the player's current speed, which can be calculated by `rb.velocity.magnitude`, so the force applied to the enemy will be stronger when the player is moving faster.

4.3 Then we show the Power Up indicator when the player has the power up, and hide it when the power up is ended. We can achieve this by using `SetActive` method to show/hide the Power Up indicator. But first, add one indicator from `/Assets/Course Libray/Powerup Indicator` as a child of the player, and then assign it to the `powerUpIndicator` variable in the inspector.

4.4 The power up is not last forever, we need to reset the power up after some time, so that the player can get the power up again. We can achieve this by using `Invoke` method to call a method to reset the power up after some time and also hide the Power Up indicator.

```c#
    void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("PowerUp"))
        {
            hasPowerup = true;
            Destroy(other.gameObject);
            Invoke(nameof(ResetPowerup), 5f);
        }
    }

    void ResetPowerup()
    {
        hasPowerup = false;
        Debug.Log("Powerup has ended");
        powerUpIndicator.SetActive(false);
    }
```

4.5 Or we can also use Coroutine to reset the power up after some time, which is more flexible and can be easily extended to add some cool effect when the power up is active (e.g. change the color of the player, or add some particle effect, etc.)

```c#
    void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("PowerUp"))
        {
            hasPowerup = true;
            Destroy(other.gameObject);
            StartCoroutine(PowerupCooldown());
        }
    }

    IEnumerator PowerupCooldown()
    {
        powerUpIndicator.SetActive(true);
        yield return new WaitForSeconds(50f);
        hasPowerup = false;
        Debug.Log("Powerup has ended");
        powerUpIndicator.SetActive(false);
    }
```
